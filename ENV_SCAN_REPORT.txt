================================================================================
  דוח מצב — ניהול משתני סביבה (.env) אחרי שדרוג
  עדכון: 2026-03-23
================================================================================

1. פורמט אחיד
-------------
כל הקבצים משתמשים בפורמט KEY=value (ללא רווחים סביב =).

2. מקור אמת
------------
.env.example — תבנית מלאה עם הערות: חובה / אופציונלי / MongoDB / Legacy.

3. אסטרטגיית MongoDB
--------------------
- ראשי: MONGODB_URI (מחרוזת מלאה, מומלץ ל-Vercel).
- חלופה: DB_NAME + DB_PASSWORD + MONGODB_CLUSTER_HOST.
- Legacy בקוד בלבד: MONGO_URI (תאימות לאחור) — ראה utils/mongoConnectionStrings.js.

4. התאמת קוד
-------------
- users/models/usersAccessDataService.js: DB נקרא מ-config.get("DB") כמו ב-cards.
- config/custom-environment-variables.json: מיפוי מלא ל-NODE_ENV, PORT, DB, JWT,
  MongoDB, TOKEN_GENERATOR, LOGGER.

5. קבצים מקומיים (לא ב-Git)
-----------------------------
.env.development — כולל TOKEN_GENERATOR, LOGGER, MONGODB_URI, MONGODB_CLUSTER_HOST.
.env.production — אוחד לפורמט; אם הוחלפו סודות ב-placeholder — יש להחזיר מ-Vercel או מגיבוי.

6. נוהל המשך
--------------
כל משתנה חדש: קוד + .env.example + סביבה מקומית + Vercel (אם פרודקשן).
לפני push: git status — ללא .env עם סודות.
